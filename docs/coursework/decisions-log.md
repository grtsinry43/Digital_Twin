# 决策记录

记录在课设推进过程中做的关键路线选择。按时间顺序。

## Decision 1 · 采用 A+ 方案（介于 A 和 B 之间）

**日期**：课设初期

**问题**：原仓 `Digital_Twin` 主循环强依赖物理系统 + MQTT broker + 两进程通信；我们没有物理线路，也没时间搭完整 MQTT 管道。

**三种选项**

| 选项 | 说明 | 工期 | 论文忠实度 |
|---|---|---|---|
| A · 只跑数字模型 | 只用 `Model` 类做纯仿真，不涉及同步/MQTT | 短 | 只覆盖算法层 |
| B · 完整 DT 系统 | 装 mosquitto，写物理模拟器 + 合成 `real_log`，完整主循环 | 2-3 天 | 高 |
| A+ · A 为主 + 伪 MQTT | A 方案核心 + 把单进程事件流在 UI 上呈现为"双向通信" | 中 | 覆盖算法 + 通信概念 |

**选择**：A+

**理由**：
- 论文核心贡献是层 2（RCT 预测驱动路径决策），A+ 完整覆盖
- 课程主题"工业互联网"强调通信，A+ 在 UI 层补回这一点
- 工期可控，风险低
- 答辩时能大方讲清"物理系统用 SimPy 仿真替代"这个工程折中

## Decision 2 · Case02 用"同步贪心"而非"纯贪心"

**日期**：选 A+ 之后，开工前

**问题**：A+ 下 Case02（启用 RCT 服务）有三种实现粒度。

**三种选项**

| 方案 | 原理 | 数字孪生完整度 | 工期 |
|---|---|---|---|
| C2-简化 | 静态策略（总是送到更快机器 M4） | ❌ 无预测 | 1h |
| C2-贪心 | t=0 时用 twin 一次性预测每个初始零件的最优路径 | ⚠️ 有预测，无同步 | 半天 |
| C2-同步贪心 | 主仿真每 Δt 暂停 → 孪生快照 → 逐路径 RCT 预测 → 写回 | ✅ 完整 | 1 天 |
| C2-完整 | 保留原 orchestrator 全部模块 | ✅ 完整 | 2-3 天 |

**选择**：C2-同步贪心

**理由**：
- 用户在被问及"贪心有数字孪生么"时指出：严格定义里孪生要有"从物理系统同步状态"的能力
- 纯贪心缺"同步"这一环，答辩时会被抠
- 同步贪心在主仿真每隔 Δt 暂停、从主仿真状态构造孪生、跑 RCT 预测，**主仿真 = 物理系统、孪生 = 数字孪生**，完整的四层能力都在
- 比 C2-完整省掉 MQTT 与两进程通信，但保留了论文的算法核心与同步概念

**代价**：SimPy 无法 deepcopy（generator 对象），需要自己写"状态快照 → JSON → 重建 Model"的管道，这正是原仓 Synchronizer 在物理系统侧做的事——我们把它内化到内存。

## Decision 3 · 决策扫描策略：`scan_all=True`

**日期**：Case02 首次运行后

**问题**：初版严格按论文只对 `queue_position=2`（队列里第二个）的零件做决策。2000 时间单位下只触发 8 次决策，Case02 的改善幅度不及论文数据。

**选择**：每次检查所有分岔队列里所有未决策零件（`scan_all=True`）

**理由**：
- 论文的 `queue_position=2` 设计是为了给物理系统留缓冲时间，在"纯数字"场景里不存在这个约束
- 扫所有零件后决策数从 8 提升到 28，Case02 Max CT 从 540 降到 412，数据明显改善
- 仍然严格使用 RCT 预测做决策，不是启发式

**影响**：答辩讲稿里需要注明"我们的决策扫描比论文更激进，因为没有物理系统通信延迟约束"。

## Decision 4 · 状态快照的近似处理

**日期**：写 `twin_sync.py` 时

**问题**：SimPy 主仿真中途暂停时，部分零件可能：
1. 在某个队列里 — 容易快照
2. 正在某台机器里加工 — 需要记录剩余加工时间
3. 在某条传送带上 — 需要记录剩余运输时间

严格快照意味着要扩展 Model 接收"机器已加工 X 时间 / 传送带剩余 Y 时间"的初始状态，工程量大。

**选择**：把 (2)(3) 里的零件近似放入最近的队列

- 机器内加工中的零件 → 回到机器的输入队列
- 传送带上的零件 → 前进到传送带的目标队列

**理由**：
- 对整体 RCT 的影响通常 < 单次 process_time（本模型最大 60，RCT 规模 300+），可接受
- RCT 比较是**相对**的（哪条路径更快），绝对误差在两条路径间会相互抵消
- 可以先用近似拿到可跑的结果，未来有需要再加严格版

**答辩说明**：这是工程折中，可以在文档里列为"简化假设"。
